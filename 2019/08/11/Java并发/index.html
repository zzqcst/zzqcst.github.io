Java中线程的状态

Thread类中有一个枚举类State，里面定义了线程的5中状态。

NEW

新建，创建后尚未启动。

RUNNABLE

可能正在运行，也可能正在等待 CPU 时间片。

包含了操作系统线程状态中的 Running 和 Ready。

BLOCKED

等待获取一个排它锁，如果其线程释放了锁就会结束此状态。

WAITING

等待其它线程显式地唤醒，否则不会被分配 CPU 时间片。

TIMED_WAITING

无需等待其它线程显式地唤醒，在一定时间之后会被系统自动唤醒。

TERMINATED

可以是线程结束任务之后自己结束，或者产生了异常而结束。

wait()函数

Object类中的函数，当一个线程调用一个共享变量的wait()方法时，该调用线程会被阻塞，直到（1）其它线程调用了该共享变量的notify()或者notifyAll()方法；（2）其它线程调用了该线程的interrupt()方法，该线程抛出InterruptedException异常返回。

如果调用wait()方法的线程没有获得该对象的监视器锁，则会抛出IllegalMonitorStateException异常。

当前线程调用共享变量的wait()方法后，会释放共享变量上的锁。

防止虚假唤醒：使用循环不停测试被唤醒的条件是否满足，不满足则继续等待。

join方法

等待线程执行终止。线程A内调用了线程B的join方法，线程A会被阻塞，直到线程B执行完成。

sleep方法

Thread类中的静态方法。当一个执行中的线程调用了Thread的sleep方法后，调用线程会让出指定时间的执行期，在此期间不参与CPU的调度，但是该线程所拥有的监视器资源，比如锁还是持有不让出的。

yield方法

Thread类中的一个静态方法，当一个线程调用yield方法时，当前线程会让出CPU使用权，然后进入就绪状态，线程调度器就会从就绪队列中获取一个线程来运行，也有可能会调度到刚刚让出CPU时间的线程。

总结：sleep和yield的区别在于，当线程调用sleep方法时调用线程会被阻塞挂起。而调用yield方法时，线程只是让出自己剩余的时间片，并没有被阻塞挂起，而是出于就绪状态。

线程中断

  • void interrupt()方法：线程B调用线程A的interrupt()方法来设置A的中断标志为true。但仅仅是设置标志，线程A并没有被中断，它会继续往下执行。
  • boolean isInterrupted()方法：检测当前线程是否被中断。
  • booleand interrupted()方法：与isInterrupted作用相同，不同的是，该方法如果发现当前线程被中断，则会清楚中断标记。该方法是静态方法。获取的是当前所在线程的中断状态，而不是调用该方法的实例线程的中断状态。

ThreadLocal

如果创建了一个ThreadLocal变量，那么访问这个变量的每个线程都会有这个变量的一个本地副本。

死锁例子

死锁产生的四个条件：

  1. 互斥条件：即资源同时只能由一个线程使用
  2. 请求并持有：指一个线程已经持有了至少一个资源，但又提出了新的资源请求，而新资源被其它线程持有，所以当前线程会被阻塞，但不释放自己持有的资源。
  3. 不可剥夺条件：指线程获取到的资源在自己使用完之前不能被其它线程抢占，只有在自己使用完之后才能由自己释放资源。
  4. 环路等待条件：指在发生死锁时，必然存在进程-资源的环形链，第一个线程等待第二个线程的资源，第二个等第三个，。。，第n个等第一个的。
