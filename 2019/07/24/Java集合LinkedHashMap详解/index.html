Java集合LinkedHashMap详解

发表于 2019-07-24 更新于 2019-07-25 分类于 Java集合

Map框架

简介

LinkedHashMap是实现了Map接口的哈希表和链表，具有可预测的迭代顺序。与HashMap的不同之处在于，它维护一个所有条目的双向链表。这个链表定义了迭代顺序，通常是键插入的顺序。注意，如果一个键被重新插入到映射中，插入顺序不会受到影响。

与HashMap关系

LinkedHashMap直接继承于HashMap,允许key和value为null，因为put键相同的值会覆盖之前的值，所以只会存在一个key为null。一些主要的方法和属性也直接继承自HashMap，并对其中某些方法进行了重写。LinkedHashMap可以按照插入序 (insertion-order)或访问序 (access-order)来对哈希表中的元素进行遍历。默认是按插入顺序排序，如果指定按访问顺序排序，那么调用get方法后，会将这次访问的元素移至链表尾部。

源码分析

LinkedHashMap改写了HashMap中的节点类Node，增加了before和after节点来实现双向链表。注意与next节点的区别，next是桶中的节点的下一个节点，而before和after是按插入顺序的上一个节点和下一个节点。（在1.7中使用的是环形双向链表）。

成员参数

LinkedHashMap中保存了双向链表的头节点和尾节点

put方法

LinkedHashMap没有重写父类的put方法，所以在put时，直接调用的是父类的put方法。LinkedHashMap的put过程和HashMap大致相同，包括计算hash值、计算table数组索引、判断数组是否为空等步骤，不同的是创建节点的过程。在put时，HashMap中会调用新建节点的方法Node<K,V> newNode(int hash, K key, V value, Node<K,V> next),LinkedHashMap重写了这个方法，将新增的节点添加到链表尾部。

红黑树转换

在HashMap中，如果一个桶中的链表达到条件后，会转化成红黑树存储。LinkedHashMap重写了构建和替换红黑树节点的方法，在其中加入了对双向链表的操作。

访问顺序

LinkedHashMap默认按插入顺序遍历，访问顺序通过accessOrder字段控制。提供了可以指定accessOrder值的构造方法。

在HashMap中定义了三个回调方法，LinkedHashMap实现了这些方法，在调用父类的put和remove方法时，会调用LinkedHashMap自身的这些回调方法，从而实现了访问序。

遍历

LinkedHashMap中定义的迭代器是从head头节点开始遍历,而HashMap中的迭代器是根据table数组来遍历。

使用LinkedHashMap实现LRU

总结

LinkedHashMap与HashMap几乎相同，只是增加了双向链表的结构，从而达到能够按照插入顺序访问元素的功能，在实现上，通过HashMap中的回调方法完成了对双向链表的操作，这一思路值得借鉴。
